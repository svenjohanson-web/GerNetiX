# Leitprinzipien

## Leitsatz

> Wissen ist teurer als Code.

## Grundannahmen

- Code ist austauschbar.
- Nicht Code ist das eigentliche Kapital, sondern Engineering-Wissen.
- Engineering-Wissen muss dauerhaft erhalten bleiben.
- Code implementiert Wissen.
- Das Metamodell bewahrt Wissen.
- Die Implementierung ist eine konkrete Auspraegung des Engineering-Wissens.

## Konsequenzen

- Es wird nicht direkt im Code begonnen.
- Jede Aenderung beginnt fachlich.
- Jede Implementierung muss auf einen fachlichen Nutzen zurueckgefuehrt werden koennen.
- Das Metamodell ist wichtiger als die Implementierung.
- Wissen muss erhalten bleiben, auch wenn sich Technik und Code aendern.
- Jede technische Entscheidung soll spaeter nachvollziehbar bleiben.

## Fragen an jedes Artefakt

Jedes Artefakt soll folgende Fragen beantworten koennen:

- Warum existiere ich?
- Wem gehoere ich fachlich?
- Welchen Nutzen erfuelle ich?
- Wovon haenge ich ab?
- Wer nutzt mich?
- Wofuer werde ich verwendet?

## Validierungsregel

Kein nicht-root Artefakt ohne nachvollziehbare Upstream-Beziehung.

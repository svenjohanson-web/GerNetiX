# Browser-App-Runtime fuer Tamagotchi

Diese Runtime zeigt das Tamagotchi als Browser App auf Mac, PC und Mobile.

Der Browser-Pfad ist in Lektionen unterteilt:

1. Code ist schwer verstaendlich.
2. Ein Modell beschreibt Verhalten fachlicher und lesbarer.
3. Das Modell ist unabhaengig von der Runtime.
4. Aus dem Modell koennen verschiedene Runtime-Apps entstehen.
5. Die erste erzeugte App ist eine Browser App.
6. Danach werden Vorteile und Nachteile der Browser App sichtbar gemacht.

Die erste Browser App darf bewusst einfach sein. Sie reagiert nur auf Benutzerinteraktionen und speichert noch nichts. Dadurch wird der Nachteil sichtbar: Browser schliessen bedeutet, dass das Tamagotchi weg ist.

In der naechsten Lektion wird dieselbe App erweitert:

- zeitgesteuerte State-Machine
- regelmaessiger Tick
- Speichern und Laden von Modell und Zustand
- Vergleich zwischen Event-Logik und Zeit-Logik
